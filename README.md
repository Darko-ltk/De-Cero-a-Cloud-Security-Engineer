```markdown
# Día 1: Configuración del Entorno de Laboratorio

**Objetivos del Día:**
- Instalar el software de virtualización necesario (VirtualBox).
- Descargar la imagen ISO oficial de Ubuntu Server 22.04 LTS.
- Crear y configurar la primera máquina virtual que será nuestro servidor base.

---

### 1. Instalación de VirtualBox

VirtualBox es un software de virtualización gratuito y de código abierto que nos permitirá crear máquinas virtuales (VMs) en nuestro sistema operativo actual.

- **Acción:** Ve a la [página oficial de descargas de VirtualBox](https://www.virtualbox.org/wiki/Downloads).
- **Instrucciones:** Descarga el paquete correspondiente a tu sistema operativo (Windows, macOS, Linux) e instálalo siguiendo el asistente. Asegúrate de instalar también el "VirtualBox Oracle VM VirtualBox Extension Pack" que se encuentra en la misma página, ya que mejora la compatibilidad y añade funcionalidades.

### 2. Descarga de Ubuntu Server 22.04 LTS

LTS (Long-Term Support) significa que esta versión de Ubuntu tendrá soporte y actualizaciones de seguridad durante 5 años, lo que la hace ideal para servidores.

- **Acción:** Ve a la [página oficial de descargas de Ubuntu Server](https://ubuntu.com/download/server).
- **Instrucciones:** Descarga la imagen ISO de **Ubuntu Server 22.04 LTS**. Es un archivo que pesa varios gigabytes, así que ten paciencia.

### 3. Creación de la Primera Máquina Virtual

Ahora vamos a crear el "ordenador virtual" donde instalaremos nuestro servidor.

- **Acción:** Abre VirtualBox y haz clic en "Nueva".
- **Configuración recomendada:**
    - **Nombre:** `hacklab-server-01`
    - **Tipo:** `Linux`
    - **Versión:** `Ubuntu (64-bit)`
    - **Memoria RAM:** `4096 MB` (4GB)
    - **Disco Duro:** Crear un disco virtual ahora.
    - **Tipo de archivo de disco duro:** `VDI (VirtualBox Disk Image)`
    - **Almacenamiento:** `Reservado dinámicamente`
    - **Tamaño del disco:** `50 GB`

Una vez creada, selecciona la VM y ve a **Configuración > Red**. Asegúrate de que el Adaptador 1 esté conectado a `Adaptador Puente (Bridged Adapter)`. Esto permitirá que tu VM obtenga una IP de tu red local, como si fuera un ordenador físico más.

---

**Fin del Día 1:** ¡Felicidades! Has montado la base de tu laboratorio. Mañana instalaremos el sistema operativo en esta nueva máquina virtual.
```

```markdown
# Día 3: Administración Básica del Sistema

**Objetivos del Día:**
- Comprender la estructura de directorios de Linux (Filesystem Hierarchy Standard).
- Dominar los comandos básicos de navegación y manipulación de directorios.
- Aprender a gestionar archivos: crear, copiar, mover, renombrar y eliminar.

---

### 1. Estructura del Sistema de Ficheros de Linux

A diferencia de Windows, en Linux todo parte de un único directorio raíz (`/`). Dentro de él, hay directorios estándar con propósitos específicos. Conéctate a tu servidor por SSH para realizar las siguientes acciones.

- `/bin`, `/sbin`: Comandos esenciales para el sistema.
- `/etc`: Archivos de configuración de todo el sistema.
- `/home`: Directorios personales de los usuarios.
- `/var`: Datos variables, como logs (`/var/log`).
- `/tmp`: Archivos temporales.
- `/usr`: Programas y datos de usuario.

### 2. Navegación y Gestión de Directorios

- `pwd` (Print Working Directory): Muestra en qué directorio te encuentras.
  ```
  pwd
  ```
- `ls` (List): Lista el contenido de un directorio.
  - `ls -l`: Formato largo (permisos, propietario, tamaño).
  - `ls -a`: Muestra archivos ocultos (los que empiezan por `.`).
  - `ls -la`: Combina ambos.
- `cd` (Change Directory): Cambia de directorio.
  - `cd /etc`: Ve al directorio de configuraciones.
  - `cd ..`: Sube un nivel.
  - `cd ~` o `cd`: Vuelve a tu directorio home.
- `mkdir` (Make Directory): Crea un nuevo directorio.
  ```
  mkdir mis_proyectos
  ```
- `rmdir` (Remove Directory): Borra un directorio **vacío**.
  ```
  rmdir mis_proyectos
  ```

### 3. Gestión de Archivos

- `touch`: Crea un archivo vacío.
  ```
  touch mi_archivo.txt
  ```
- `cp` (Copy): Copia archivos o directorios.
  ```
  cp mi_archivo.txt mi_copia.txt
  ```
- `mv` (Move): Mueve o renombra un archivo.
  ```
  mv mi_copia.txt archivo_renombrado.txt
  mv archivo_renombrado.txt mis_proyectos/
  ```
- `rm` (Remove): Elimina archivos. **¡CUIDADO, NO HAY PAPELERA DE RECICLAJE!**
  - `rm mi_archivo.txt`
  - `rm -r mis_proyectos`: Borra un directorio y todo su contenido (`-r` de recursivo).
- `find`: Busca archivos.
  ```
  find /etc -name "ssh_config"
  ```
- `cat`, `less`, `more`: Visualizan el contenido de un archivo.
  ```
  cat /etc/passwd
  less /var/log/syslog
  ```

---

**Fin del Día 3:** Ya te mueves con fluidez por el sistema de archivos de Linux. Has aprendido los comandos que usarás a diario durante toda tu carrera.
```

```markdown
# Día 4: Permisos y Usuarios Básicos

**Objetivos del Día:**
- Entender el sistema de permisos de Linux (lectura, escritura, ejecución).
- Utilizar los comandos `chmod`, `chown` y `chgrp` para gestionar permisos.
- Aprender a crear y gestionar usuarios y grupos básicos.

---

### 1. El Sistema de Permisos de Linux

Cuando ejecutas `ls -l`, ves algo como `-rwxr-xr--`. Esto representa los permisos.

- **El primer carácter:** `-` para archivo, `d` para directorio.
- **Los siguientes 9 caracteres se dividen en 3 grupos de 3:**
    - **Grupo 1 (rwx):** Permisos para el **Usuario** propietario.
    - **Grupo 2 (r-x):** Permisos para el **Grupo** propietario.
    - **Grupo 3 (r--):** Permisos para **Otros** (todos los demás).
- **Significado de las letras:**
    - `r`: **Read** (Leer)
    - `w`: **Write** (Escribir)
    - `x`: **Execute** (Ejecutar)

### 2. Gestión de Permisos

- `chmod` (Change Mode): Cambia los permisos de un archivo/directorio.
    - **Modo simbólico (letras):**
      ```
      chmod u+x mi_script.sh  # Añade permiso de ejecución (x) al usuario (u).
      chmod g-w archivo.txt   # Quita permiso de escritura (w) al grupo (g).
      chmod o=r archivo.txt   # Establece solo permiso de lectura (r) para otros (o).
      chmod a+r archivo.txt   # Añade lectura para todos (a).
      ```
    - **Modo octal (números):** `r=4`, `w=2`, `x=1`. Se suman.
      ```
      chmod 754 mi_script.sh # rwx r-x r-- (7=4+2+1, 5=4+1, 4=4)
      ```
- `chown` (Change Owner): Cambia el propietario (usuario y grupo).
  ```
  sudo chown nuevo_usuario:nuevo_grupo mi_archivo.txt
  ```
- `chgrp` (Change Group): Cambia solo el grupo propietario.
  ```
  sudo chgrp nuevo_grupo mi_archivo.txt
  ```

### 3. Gestión de Usuarios y Grupos

- `sudo useradd -m nombre_usuario`: Crea un nuevo usuario y su directorio home (`-m`).
- `sudo passwd nombre_usuario`: Establece una contraseña para el nuevo usuario.
- `sudo usermod -aG grupo usuario`: Añade un usuario a un grupo existente (`-aG` para Append to Group). Muy común para añadir usuarios al grupo `sudo`.
- `sudo userdel -r nombre_usuario`: Elimina un usuario y su directorio home (`-r`).
- `sudo groupadd nombre_grupo`: Crea un nuevo grupo.
- `cat /etc/passwd`: Muestra la lista de usuarios.
- `cat /etc/group`: Muestra la lista de grupos.

---

**Fin del Día 4:** Has aprendido a controlar quién puede hacer qué en tu sistema. La gestión de permisos y usuarios es la base de la seguridad en Linux.
```

```markdown
# Día 5: Administración de Paquetes con APT

**Objetivos del Día:**
- Entender el concepto de repositorios y gestión de paquetes.
- Dominar el uso de `apt` para instalar, actualizar y eliminar software.
- Aprender a buscar paquetes y gestionar los repositorios del sistema.

---

### 1. ¿Qué es un Gestor de Paquetes?

En lugar de descargar instaladores desde páginas web, los sistemas Linux utilizan gestores de paquetes que se conectan a servidores centrales (llamados **repositorios**) para descargar, instalar y gestionar software de forma segura y controlada. En Ubuntu (y Debian), la herramienta principal es `APT` (Advanced Package Tool).

### 2. Comandos Esenciales de `apt`

Todos estos comandos requieren privilegios de administrador, por lo que se ejecutan con `sudo`.

- `sudo apt update`: **El primer comando que siempre debes ejecutar.** Sincroniza la lista de paquetes disponibles desde los repositorios. No instala nada, solo actualiza la "base de datos" de lo que está disponible.
- `sudo apt upgrade`: Compara los paquetes instalados con la lista actualizada y actualiza todos los que tengan una nueva versión.
- `sudo apt install <nombre_del_paquete>`: Instala un paquete nuevo.
  ```
  sudo apt install htop  # htop es un monitor de sistema muy útil.
  ```
- `sudo apt remove <nombre_del_paquete>`: Desinstala un paquete, pero puede dejar archivos de configuración.
- `sudo apt purge <nombre_del_paquete>`: Desinstala un paquete y también borra sus archivos de configuración.
- `sudo apt autoremove`: Elimina paquetes que se instalaron como dependencias de otros pero que ya no son necesarios. Es bueno ejecutarlo después de desinstalar software.

### 3. Búsqueda y Gestión

- `apt search <término_de_búsqueda>`: Busca paquetes en los repositorios.
  ```
  apt search firewall
  ```
- `apt show <nombre_del_paquete>`: Muestra información detallada sobre un paquete (versión, dependencias, descripción).
  ```
  apt show nginx
  ```
- `dpkg -l`: Lista todos los paquetes instalados en el sistema. Puedes filtrar la salida con `grep`: `dpkg -l | grep htop`.

### 4. Gestión de Repositorios

Los repositorios que `apt` utiliza están definidos en el archivo `/etc/apt/sources.list` y en los archivos dentro del directorio `/etc/apt/sources.list.d/`. Por ahora, no es necesario modificarlos, pero es importante saber dónde se encuentran.

---
```

```markdown
# Día 6: Servicios del Sistema con systemd

**Objetivos del Día:**
- Entender qué es un servicio (o demonio) en Linux.
- Aprender a gestionar servicios con `systemctl`, la herramienta principal de `systemd`.
- Habilitar y deshabilitar servicios para que se inicien (o no) con el sistema.
- Revisar los logs del sistema y de servicios específicos con `journalctl`.

---

### 1. ¿Qué es `systemd`?

`systemd` es el sistema de inicio y gestor de servicios en la mayoría de las distribuciones modernas de Linux, incluyendo Ubuntu. Es el primer proceso que se ejecuta (PID 1) y se encarga de iniciar todos los demás procesos y servicios del sistema. Un "servicio" es un programa que se ejecuta en segundo plano, como el servidor SSH (`sshd`), un servidor web o una base de datos.

### 2. Gestión de Servicios con `systemctl`

`systemctl` es tu navaja suiza para interactuar con los servicios.

- `sudo systemctl status <nombre_del_servicio>`: Muestra el estado detallado de un servicio. Es el comando más útil para diagnosticar problemas.
  ```
  sudo systemctl status sshd
  ```
  Verás si está `active (running)`, `inactive (dead)` o `failed`.

- `sudo systemctl start <nombre_del_servicio>`: Inicia un servicio.
- `sudo systemctl stop <nombre_del_servicio>`: Detiene un servicio.
- `sudo systemctl restart <nombre_del_servicio>`: Reinicia un servicio. Esto es necesario después de cambiar su configuración.
- `sudo systemctl reload <nombre_del_servicio>`: Recarga la configuración sin detener el servicio (no todos los servicios lo soportan).

### 3. Habilitar y Deshabilitar Servicios

- `sudo systemctl enable <nombre_del_servicio>`: Habilita un servicio para que se inicie automáticamente cada vez que el sistema arranca.
- `sudo systemctl disable <nombre_del_servicio>`: Deshabilita el inicio automático del servicio.
- `systemctl is-enabled <nombre_del_servicio>`: Comprueba si un servicio está habilitado o deshabilitado.

### 4. Revisión de Logs con `journalctl`

`systemd` tiene su propio sistema de logging centralizado, gestionado por `journald`. La herramienta para consultarlo es `journalctl`.

- `journalctl`: Muestra todos los logs del sistema, desde el más antiguo.
- `journalctl -f`: Sigue los logs en tiempo real (muy útil para ver qué pasa mientras haces algo).
- `journalctl -u <nombre_del_servicio>`: Muestra los logs de un servicio específico.
  ```
  journalctl -u sshd
  ```
- `journalctl -p err`: Muestra solo los mensajes de error y superiores.

---

**Fin del Día 6:** Ahora tienes el control sobre los procesos que se ejecutan en tu servidor. Sabes cómo iniciarlos, detenerlos y, lo más importante, cómo ver qué están haciendo a través de sus logs.
```
